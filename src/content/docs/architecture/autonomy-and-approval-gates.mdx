# Autonomy and Approval Gates

Good automation is not about removing humans from every decision. It is about moving routine work forward while keeping consequential actions supervised.

## Design goal

The goal is to preserve momentum without losing trust.

## Where automation should run

Automation is strongest when it handles:

- planning drafts
- structured analysis
- code scaffolding
- test preparation
- status summarization
- low-risk internal operations

## Where human approval stays

Humans should remain in control of:

- protected branch merges
- public publishing
- sensitive integrations
- payments or transactions
- any action with unclear blast radius

## Why this matters publicly

A public engineering portfolio becomes much stronger when it shows not only what the system can do, but where the system intentionally stops and asks for approval.
